Como funciona Factus

Factus busca simplificar la implementación de la Factura Electrónica en las empresas del Perú. Con Factus hemos resuelto todos los requerimientos técnicos de la SUNAT respecto a la generación de los comprobantes electrónicos. (XML UBL 2.0, Xmldsig, PDF417, etc...)

Si tienes un sofware de ventas, puedes implementar Factus en tiempo record. Para simplificar la integración con las aplicaciones hemos creado dos tipos de componentes:

  • Factus API Desktop.
  • Factus API REST.

Para resolver el tema de la conservación de los comprobantes, hemos creado Factus Web (factus.pe), ambos componentes de integración envian los comprobantes a Factus Web y es este quien se encarga de asegurar el envío del comprobante a la SUNAT, además generar y enviar los resumenes de boletas y conservarlos.

Factus API Desktop

Es una librería para Windows que permite integrarse con cualquier lenguaje de programación que sorporte la técnologia COM - DLL.

¿Quieres generar tus comprobantes durante el día y enviarlos en la noche?. Factus API Desktop es lo que necesitas, no requiere estar en línea y es una excelente alternativa para lo puntos de ventas. ¿Quieres intentarlo?, solo te tomará un par de minutos:

Generar Factura

Es realmente simple y a la vez potente, nuestra librería hace todo esto:

  • Genera el XML del comprobante basado en la especificación UBL 2.0. La generación es totalmente configurable mediante el lenguaje de plantillas Apache Velocity.
  • Genera el código de Barra PDF417 según las especificaciones de la SUNAT, en cuanto a tamaño, parametros y contenido.
Código de barra PDF417
  • Firma el XML del comprobante con el certificado digital del negocio. La inclusión del certificado es totalmente configurable vía interface gráfica. (Nuesta API además te alerta sobre las fechas de caducidad del certificado).
Pantalla de configuración
  • Genera representaciones impresas (Reporte) y es totalmente configurable utilizando iReport.
  • Realiza validaciones al documento antes de generarlo. Valida contra el XSD, valida el PrecioUnitario x Cantidad, los números de RUC, etc. Y si quieres reducir el código permite configurar el emisor por defecto, así indicarle que genere el monto en letras, entre otras cosas y todo configurable desde una interface gráfica.
  • Esto es lo mejor, tiene un mecanismo de encolamiento que permite enviar los comprobantes cuando encuentre conexión a Internet. Cuenta además de una inteface de monitoreo donde puedes ver los comprobantes encolados y te alerta cuando se acerca al plazo máximo permito para el envio.

Para hacerla más simple aún, hemos creado ejemplos de uso de Factus API Desktop en lo principales lenguajes de programación para Windows y los hemos compartidos en Github. En la documentación encontrarás todos los ejemplos.
